Глосарий:

    Микросервисы -  это архитектурный подход к разработке программного обеспечения,
        при котором оно делится на небольшие независимые сервисы, отвечающие за отдельные функции.
        Они имеют чекто определимые границы и могут рпазвёртыватся, масштабироваться и обновлятся независимо друг от друга.


    Виртуальная машина — это компьютерная программа, которая имитирует другой компьютер.
        У виртуального компьютера СВОЯ операционная система и вычислительные мощности. 
        Ресурсы он «одалживает» у хоста — компьютера, на котором запущена виртуальная машина.
        Виртуалки запускаются со своей ОС внитри слоя гипервизовра в ОС хоста.


    Контейнер Docker — это виртуальный блок, служащий для запуска приложений.
        Каждый контейнер представляет собой пакет программного обеспечения,
        который содержит код, системные инструменты, библиотеки, зависимости и файлы конфигурации,
        необходимые для запуска определённого приложения. 
        Запуск контейнеров в Docker Engine в ОС хоста.


    В чем разница LXC и Docker:
        Docker - это контейнер для упаковки одного процесса или службы;на практике Docker это пачка легких контейнеров для упаковки одного веб-сервиса;
        LXC (Linux Container) - это легкий контейнер на один веб-сервис или сайт, включающий все службы, которые нужны для его функционирования.


    Docker Engine - программное обеспечение, которое позволяет создавать, разворачивать и администрировать контейнеризированные приложения.
        Включает:
            - Docker Daemon — сервер, который управляет объектами Docker, такими как образы, контейнеры, сети и тома.
            - REST API — интерфейс, с помощью которого программы взаимодействуют с демоном.
            - Docker CLI — клиентская утилита командной строки, через которую пользователи взаимодействуют с Docker.


    Docker Hub — это облачное хранилище образов контейнеров, предоставляемое компанией Docker.


    Dockerfile - текстовый файл, в который помещаются команды для сборки образа. Существует оптимизация для сборки в виде кеша сборки.


    Образы:

        Docker-образ — это шаблон (физически — исполняемый пакет), из которого создаются Docker-контейнеры.
            Образ хранит в себе всё необходимое для запуска приложения, помещённого в контейнер:
                -код,
                -среду выполнения,
                -библиотеки,
                -переменные окружения,
                -конфигурационные файлы.
            Docker-образ создаётся с помощью команды docker build,
            которая считывает конфигурацию создаваемого образа из специального конфигурационного файла — dockerfile.

        Особенности сборки образа:
            - Контрольная сумма (ДАЙДЖЕСТ) слоя, упорядоченная определенным образом для сборки образа - объект конфигурации.
            - Копирование при записи - механизм позволяющий создовать копию образа, только если образ был изменён.
            - Контейнер создает тонкий слой чтения и записи поверх образа.

    Драйверы хранилища (малый набор данных) и тома (большой набор данных):

        Драйверы хранилища - ПО, для записи данных на слой чтения и записи в контейнере Docker (обычно используют Файловую систему overlay2)
            Особенности: при удалинии контейнера записанные данные удалятся.
        
        Тома - просто физическая файловая система на хосте, подключенная к файловой системе Docker, 
            так что любые изменения в контейнерах записываются в хост-систему и наоборот.

            Доп. понятия:
                backing filesystem - файловая система для контейнеров Docker.
                bind mount - вмонтирование каталогов с хоста.
                tmpfs - временная файловая система в памяти хост-системы. (Файлы удаляются после отсановки контейнера)

    Модель Container Network и сетевой мост Docker:

        Container Network Model (CNM) - архитектура, предоставляющая простую и эффективную сетевую модель,
            которая позволяет запускать приложения в защищенных портативных контейнерах.

            Состав:
                - Песочница - полная конфигурация сетевого стека контейнера (обёртка).
                - Конечные точки - порты для внешнего подключения к песочинце.
                - Сеть - сложная инфраструктура, созданная пользователем, мост Linux или VLAN.

            Драйверы CNM - часть ПО, которое поддерживает корректную работу сетей Docker
                Виды:
                    -Собственные (встроенные) сетевые драйверы - часть Docker Engine.
                    -Сторонние (внешние) сетевые драйверы - подключаются к хосту Docker, как плагины. 

        Сетевой мост Docker - устройство канального уровня, используемое как механизм передачи трафика между сегментами сети.
